# Sprout OS — Security Checklist

---

## HTTP Security Headers

- [ ] `Strict-Transport-Security` — `max-age >= 31536000; includeSubDomains`
- [ ] `X-Content-Type-Options: nosniff`
- [ ] `X-Frame-Options: DENY` **or** CSP `frame-ancestors 'none'`
- [ ] `Referrer-Policy: strict-origin-when-cross-origin` (or stricter)
- [ ] `Content-Security-Policy` set and does not allow `unsafe-inline` on scripts
- [ ] `Permissions-Policy` restricts geolocation, camera, microphone

---

## Transport

- [ ] HTTP → HTTPS redirect returns 301/308
- [ ] TLS 1.2+ only (no TLS 1.0 / 1.1)
- [ ] Valid certificate, not near expiry (< 30 days)
- [ ] `www` and apex both resolve correctly to the same origin

---

## Version / Platform Leakage

- [ ] `Server` header does not expose version number
- [ ] `X-Powered-By` absent or generic
- [ ] No debug / error stack traces in production API responses
- [ ] Next.js version not exposed in response headers

---

## Authentication

- [ ] Rate limiting active on `/api/auth/login`, `/api/auth/signup`, `/api/auth/forgot-password`
- [ ] Brute-force lockout or CAPTCHA after N failed login attempts
- [ ] Passwords hashed with bcrypt / argon2 (never MD5 / SHA-1)
- [ ] Password reset tokens single-use: second click returns "link expired"
- [ ] Password reset tokens expire within 1 hour
- [ ] Google OAuth: `state` parameter verified on callback (CSRF protection)
- [ ] Session cookies: `HttpOnly; Secure; SameSite=Lax` or stricter
- [ ] Session expiry enforced server-side (not just by cookie expiry)
- [ ] Email verification required before full account access

---

## Authorization

- [ ] Unauthenticated users cannot access `/api/projects`, `/api/team`, `/api/manage/*`, etc.
- [ ] Workspace A members cannot read or modify Workspace B data (IDOR check)
- [ ] Admin-only API routes (`/api/admin/*`) reject non-admin tokens with 403
- [ ] Role checks enforced on every API route (not just UI visibility)
- [ ] Designer cannot call `/api/billing` — returns 403
- [ ] Developer cannot call design-mutation endpoints — returns 403
- [ ] Client cannot call team management endpoints — returns 403
- [ ] Viewer cannot call comment-creation endpoints — returns 403
- [ ] Plan-gated export: `/api/export/react` rejects FREE/STARTER plans with 403

---

## Input Handling

- [ ] All user input validated server-side (not only client-side)
- [ ] MongoDB queries use Mongoose schema types — no raw string concatenation
- [ ] XSS: user-generated content (project names, brief text, comments) escaped on output
- [ ] `dangerouslySetInnerHTML` not used with unsanitized user input
- [ ] File uploads: file type validated by content (magic bytes), not only by extension
- [ ] File upload size limits enforced server-side (15MB for avatars; relevant limits for PDFs)
- [ ] URL inputs (crawl, reference sites, Manage Mode site URL) validated before use — no SSRF
- [ ] JSON APIs verify `Content-Type: application/json` header on POST/PATCH requests

---

## Manage Mode — Site Credentials

- [ ] WordPress Application Passwords stored encrypted in MongoDB (not plaintext)
- [ ] Credentials never returned in API responses (even to the authenticated user)
- [ ] Credentials never logged in server logs or MCP client debug output
- [ ] Disconnecting a site permanently removes encrypted credentials from MongoDB
- [ ] Site URL validated (must be a valid domain) before attempting MCP connection
- [ ] MCP endpoint discovery response not trusted blindly — validate it matches the declared site domain

---

## Manage Mode — MCP Execution

- [ ] Claude system prompt does not allow AI-originated instructions to override safety rules (prompt injection hardening)
- [ ] PHP code executed via `sprout-execute-php` is generated by Claude — monitor for code injection patterns
- [ ] High-risk actions require approval before any MCP tool is called (not after)
- [ ] Agentic loop has a maximum iteration limit to prevent runaway tool calling
- [ ] External connector credentials (HTTP basic auth, env vars) encrypted at rest
- [ ] Stdio connector `command` and `args` validated — no shell injection via user-controlled connector config
- [ ] Webhook trigger endpoints verify HMAC signatures (not open to any POST)

---

## Build Module Security

- [ ] Module files deployed to `wp-content/sproutos-mcp-sandbox/` sandbox directory (not to arbitrary WP paths)
- [ ] Quality security gate scans for code injection, auth bypasses, and SQL injection in module code
- [ ] Released modules logged with deployer identity and timestamp (full audit trail)
- [ ] Rollback capability available for any release (not permanently applied)

---

## Secrets & Config

- [ ] No secrets in git history (`git log -p | grep -iE 'api_key|password|token|secret'` clean)
- [ ] `.env` and `qa.config.json` in `.gitignore`
- [ ] Production secrets in vault / secret manager (not in the repo)
- [ ] `NEXT_PUBLIC_` env vars contain no sensitive values (publicly visible in browser)
- [ ] OpenAI API key not exposed in client-side bundles
- [ ] MongoDB connection string not exposed in client-side bundles

---

## Token System Security

- [ ] Sprout Token deduction happens server-side (not client-side — client cannot fake tokens consumed)
- [ ] Token balance cannot go below 0 via race condition (atomic decrement or transactional check)
- [ ] AI features blocked when token balance is 0 — API returns 402, not a silent failure
- [ ] Token usage table accessible only to workspace Admin and the acting user (not other roles)

---

## Email & Invites

- [ ] Invite tokens are cryptographically random (not sequential IDs)
- [ ] Expired invite tokens return a clear error (not a generic 404 that could be probed)
- [ ] Cancelling an invite invalidates the token server-side immediately
- [ ] ZeptoMail SMTP credentials not in source code

---

## Dependencies

- [ ] `npm audit` — 0 critical / high vulnerabilities
- [ ] Dependabot / Renovate enabled for automated dependency alerts
- [ ] Third-party scripts (Mixpanel) reviewed and loaded asynchronously

---

## Monitoring

- [ ] Failed login attempts logged with IP and timestamp
- [ ] Unusual traffic patterns (brute-force, scraping) trigger alerts
- [ ] Error tracking (Sentry or similar) scrubs PII before sending
- [ ] Manage Mode: all MCP tool executions logged with user, action, site, and timestamp
- [ ] Manage Mode: approval decisions logged with reviewer identity
